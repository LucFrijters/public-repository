# Contributing to Public Repository

Thank you for your interest in contributing to this repository!

## Guidelines

### What to Contribute

This repository accepts:
- ✅ Non-sensitive configuration files
- ✅ Example configurations and templates
- ✅ Lists and arrays of software/tools
- ✅ Public, shareable settings

### What NOT to Contribute

**Never commit:**
- ❌ API keys, tokens, or credentials
- ❌ Passwords or secrets
- ❌ Private SSH keys or certificates
- ❌ Personal or sensitive data
- ❌ Internal network configurations
- ❌ Database connection strings with credentials

## How to Contribute

1. **Choose the right directory:**
   - `configs/development/` - Development tool configurations
   - `configs/system/` - System-level configurations
   - `lists/` - Data lists and arrays
   - `examples/` - Template configurations

2. **Prepare your file:**
   - Remove all sensitive information
   - Add comments explaining usage
   - Use clear, descriptive filenames
   - Include a README in new directories

3. **Submit your contribution:**
   - Fork the repository
   - Create a new branch
   - Add your files
   - Submit a pull request

## File Format Guidelines

- Use JSON for data lists and arrays
- Use appropriate file extensions
- Include inline comments where helpful
- Follow existing naming conventions

## Quality Standards

- Files should be well-formatted
- Include documentation or comments
- Test that configurations work as expected
- Ensure no hardcoded sensitive values

## Security

Before submitting, double-check that your files contain:
- No passwords or credentials
- No API keys or tokens
- No private keys or certificates
- No personal information

Thank you for helping keep this repository secure and useful!
